# 界面说明与设计规划（UI Guide & Design Plan）

本文说明 `xiaohongshu-matrices-cli` 的用户界面形态、页面导览、安全模型，
并给出后续 UI 演进的设计规划。面向运营者、二次开发者和设计协作方。

---

## 一、产品形态：两层界面

本项目提供 **两套互补的用户界面**，均面向“本机优先（local-first）”的工作方式：

| 界面 | 命令 | 形态 | 适用场景 |
| --- | --- | --- | --- |
| 命令行 | `xhs` | 终端 CLI（Click + Rich） | 脚本化、批量、自动化、被其他程序调用 |
| 本地后台 | `xhs-dashboard` | Web UI（FastAPI + Jinja2） | 可视化操作、审核流、素材管理、人设/二创 |

**启动后台：**

```bash
xhs-dashboard                 # 默认 http://127.0.0.1:8765
xhs-dashboard --port 9000     # 自定义端口（仍仅限本机）
xhs-dashboard --data-dir /path/to/data   # 自定义数据目录
```

> 安全约束：后台**只允许监听本机地址**（`127.0.0.1` / `localhost` / `::1`）。
> 跨域写请求（POST/PUT/PATCH/DELETE 来自非本机 Origin）一律被拒绝。
> 数据默认存于 `~/.xiaohongshu-cli/dashboard`（Windows：`%USERPROFILE%\.xiaohongshu-cli\dashboard`）。

## 二、后台页面导览（10 个导航）

| 路径 | 导航名 | 功能 |
| --- | --- | --- |
| `/` | 总览 | 账号 / 就绪账号 / 素材 / 待审 / 队列统计；最近搜索任务、最近发布任务 |
| `/accounts` | 账号管理 | 创建档案、扫码 / QR 登录绑定小红书账号 |
| `/searches` | 爆款搜索 | 创建采集任务，抓取爆款笔记 + 评论写入本地素材库 |
| `/library` | 素材库 | 已采集素材的浏览、检索、沉淀 |
| `/publish` | 审核与发布 | 上传 / 导入内容；**必须人工审核（pending_review）后才可执行发布** |
| `/materials` | 授权素材与二创 | 授权素材与二次创作（对接 `source_account_mappings` / `image_prompts`） |
| `/personas` | 账号人设 | 角色库（对接 `roles` / `account_roles`），人设画像与账号绑定 |
| `/research` | AI 研究 | AI 研究模块（多模态图片拆解 `image_decompose` 等） |
| `/engagement` | 互动工作台 | 互动操作工作台 |
| `/rules` | 互动规则 | 互动规则配置 |
| `/analytics` | 数据分析* | 运维 / 运营分析视图（未在主导航，供深度使用） |
| `/api_health` | API 健康* | 接口健康监控（未在主导航，供排障） |

> *`/analytics`、`/api_health` 为侧边栏之外的运维视图，按需访问。

导航侧栏常驻“仅在本机运行 + 数据目录”提示，强化本地信任心智。

## 三、安全与信任模型（UI 层表现）

后台在 UI 与接口层面落实了以下信任边界，UI 设计应**显式传达**这些约束：

- **仅本机监听**：启动参数限制 host，TrustedHost 白名单 + 跨域守卫。
- **不存明文 Cookie**：数据库不保存明文登录态；浏览器档案按平台隔离（Windows ACL 仅当前用户 + SYSTEM，其他系统 `0o700`）。
- **发布强制人工审核**：所有发布任务默认 `pending_review`，必须人工批准；同账号串行、默认间隔 ≥10 分钟。
- **断点恢复 / 防重复**：搜索候选逐篇落库，重启可恢复；发布核验要求出现新笔记 ID 才判定成功，不自动重发。

**设计建议**：在“账号管理 / 审核与发布”页面常驻状态徽标（就绪 / 验证待定 / 暂停），
让风控状态对用户可见、可解释。

## 四、UI 设计规划（Roadmap）

当前 UI 为“单 `style.css` + Jinja 服务端模板”的轻量形态，适合本机工具。
后续演进遵循“**不引入重前端栈、保持本机轻量**”原则，方向如下：

### 1. 信息架构重组
将现有 10 个页面按业务流分组，降低认知负担：

```
采集层    ：账号管理 · 爆款搜索 · 素材库
生产层    ：账号人设（角色库） · 授权素材与二创 · AI 研究
发布层    ：审核与发布
互动层    ：互动工作台 · 互动规则
系统层    ：总览 · 数据分析 · API 健康
```

### 2. 角色库 UI（对接已落地的 `roles` / `account_roles`）
- `personas` 页升级为“角色库”：创建角色（画像、语气、禁忌）、将角色绑定到账号（主/副）。
- 支持从“账号人设”一键派生二创任务的风格约束，串联 `/materials` 与 `/publish`。

### 3. 多模态拆解结果可视化（对接 `image_prompts`）
- 在 `/materials` 二创页展示 `image_decompose` 结果：主体 / 构图 / 色调 / 风格 /
  可复用 prompt / 标签，支持复制与二次编辑，沉淀为可复用素材模板。

### 4. 设计系统（Design Tokens）
- 将 `style.css` 重构为基于 CSS 变量的设计令牌（颜色 / 间距 / 圆角 / 字号），
  统一侧栏、卡片、表单、徽标样式；保证移动端 / 窄屏可读（响应式）。

### 5. 信任标识常驻
- 全局常驻“仅本机运行”徽标 + 当前数据目录；敏感操作（发布、删除）二次确认。

### 6. 轻交互增强（非 SPA）
- 在保持服务端模板的前提下，引入 HTMX / Alpine.js 做局部刷新与简单交互，
  避免引入打包构建链，维持“装上即用、本机零前端依赖”的体验。

### 7. 可访问性与暗色模式（可选）
- 提供暗色主题切换；保证对比度与键盘可达性。

## 五、CLI 速览（与 UI 互补）

CLI 覆盖搜索、阅读、互动、创作、通知、账号矩阵等能力，支持 `--json` / `--yaml` 结构化输出，
适合被脚本或 agent 调用。详细命令见 `xhs --help` 与各子命令 `--help`。
UI 负责“看与审”，CLI 负责“做与连”，二者共享同一套本地数据目录与治理规则。
